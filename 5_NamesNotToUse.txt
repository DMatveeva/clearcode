1) timeSorting -> sortingTimeMs // Время сортировки в миллисекундах.
2) value -> logisticDistributed // Случайная величина, подчиняющаяся логистическому распределению.
3) defVariance -> scaleParameter // Формула для расчета дисперсии: variance = b^2 * pi^2 /3. В моем коде множитель b назывался defVariance (defining variance) - переменная, которая определяет дисперсию. А надо было назвать ее параметром масштаба.
4) s -> variance // Изначально переменная s представляла собой корень из дисперсии. Лучше было не извлечать корень, и назвать переменную "Дисперсия".
5) result -> participantProfiles // Список результатов вычислений для участников.
6) sum -> sumOfDifferences // Суммируется попарная разность значений для чисел из 2-х файлов.
7) as -> altruisticStrategy // as ассоциируется с объявлением алиаса
8) list -> lines // Переменная хранит строки после парсинга файла.
9) logg -> logisticGenerator // Генератор случайных чисел, подчиняющихся логистическому распределению. А log - обычно логгер.
10) lg -> laplaceGenerator // Генератор случайных чисел, подчиняющихся распределению Лапласа.
11) В цикле объявляются переменные curE, curA, curG. Имеется в виду значение коэффициентов эгоизма, альтруизма и лояльности к группе для текущего участника. Легко спутать, т.к. различие только в последней букве. Лучше назвать egoismCoeff, altruismCoeff,groupCoeff например. А curr вообще опустить, т.к. очевидно, что коэффициенты относятся к участнику на текущей итерации. 
12) paramS -> parameters // Изначально было сокращение от paramString.
